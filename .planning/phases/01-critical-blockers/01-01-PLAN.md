<objective>
Convert LogViewer.csproj from single-target (net10.0-windows) to multi-target (net8.0-windows;net10.0-windows), and make the three Microsoft.Extensions.* package references version-conditional so they resolve 8.0.6 under net8 and 10.0.6 under net10.

Purpose: The NuGet package currently cannot be installed into any .NET 8 WPF project. This is the primary compatibility blocker for v1.0.0.
Output: An updated LogViewer.csproj that builds two TFM outputs from a single `dotnet build` invocation.
</objective>

  <action>
Make the following exact changes to LogViewer/LogViewer.csproj:

1. In the first `<PropertyGroup>`, change line 5 from:
   ```xml
   <TargetFramework>net10.0-windows</TargetFramework>
   ```
   to:
   ```xml
   <TargetFrameworks>net8.0-windows;net10.0-windows</TargetFrameworks>
   ```
   (element name changes from singular to plural; value gains both TFMs separated by semicolon)

2. Remove the existing flat `<PackageReference>` lines for the three Microsoft.Extensions.* packages from the shared `<ItemGroup>` (lines that reference `Microsoft.Extensions.DependencyInjection.Abstractions`, `Microsoft.Extensions.Logging`, and `Microsoft.Extensions.Logging.Abstractions` at Version="10.0.6").

3. Insert two new conditional `<ItemGroup>` blocks immediately after the shared package `<ItemGroup>`:

   ```xml
   <ItemGroup Condition="'$(TargetFramework)' == 'net8.0-windows'">
     <PackageReference Include="Microsoft.Extensions.DependencyInjection.Abstractions" Version="8.0.6" />
     <PackageReference Include="Microsoft.Extensions.Logging" Version="8.0.6" />
     <PackageReference Include="Microsoft.Extensions.Logging.Abstractions" Version="8.0.6" />
   </ItemGroup>

   <ItemGroup Condition="'$(TargetFramework)' == 'net10.0-windows'">
     <PackageReference Include="Microsoft.Extensions.DependencyInjection.Abstractions" Version="10.0.6" />
     <PackageReference Include="Microsoft.Extensions.Logging" Version="10.0.6" />
     <PackageReference Include="Microsoft.Extensions.Logging.Abstractions" Version="10.0.6" />
   </ItemGroup>
   ```

All other packages (CommunityToolkit.Mvvm 8.4.0, CsvHelper 33.1.0, Fody 6.9.2, Newtonsoft.Json 13.0.3, PropertyChanged.Fody 4.1.0) remain in the shared `<ItemGroup>` unchanged.

After editing, run:
```
dotnet build LogViewer/LogViewer.csproj --configuration Release
```
Both TFM outputs must appear in the build output. Fix any compilation errors before proceeding.
